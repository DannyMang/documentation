## Upload Dataset

<ParamField body="name" type="string" required>
  A human-readable name for the dataset (sent as a form field alongside the file).
</ParamField>

<ParamField body="file" type="file" required>
  A JSONL file where each line is a JSON object containing a `prompt` field.
</ParamField>

### Request

```bash
curl -X POST https://api.veri.studio/v1/datasets \
  -H "Authorization: Bearer vk_your_api_key" \
  -F "name=math_prompts" \
  -F "file=@math_prompts.jsonl"
```

### Response

```json
{
  "id": "ds_abc123",
  "object": "dataset",
  "name": "math_prompts",
  "source_type": "upload",
  "source_uri": null,
  "huggingface_dataset": null,
  "num_rows": 1500,
  "created_at": "2026-04-14T12:00:00Z"
}
```

## Connect Dataset

<ParamField body="name" type="string" required>
  Display name for the connected dataset.
</ParamField>

<ParamField body="source_type" type="string" required>
  One of `hf` or `volume`. (Direct file uploads use `POST /v1/datasets`, not this endpoint.)
</ParamField>

<ParamField body="huggingface_dataset" type="string">
  Required for `hf` sources. The Hugging Face dataset slug (e.g. `openai/gsm8k`).
</ParamField>

<ParamField body="huggingface_config" type="object">
  Required for `hf` sources. Object containing `config_name` (optional, for multi-config datasets), `split` (e.g. `train`), and `column_mapping` (a dict mapping source columns to `prompt` / `answer` / `label`).
</ParamField>

<ParamField body="volume" type="string">
  Required for `volume` sources. The volume name.
</ParamField>

<ParamField body="volume_path" type="string">
  Required for `volume` sources. Absolute path to the JSONL file inside the volume (e.g. `/train.jsonl`).
</ParamField>

<Note>
  Connecting two datasets with the same `name` but a different source config returns `409 dataset_name_conflict`. The same name + same config is idempotent (the existing dataset is returned).
</Note>

### Request

```bash
curl -X POST https://api.veri.studio/v1/datasets/connect \
  -H "Authorization: Bearer vk_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "gsm8k-train",
    "source_type": "hf",
    "huggingface_dataset": "gsm8k",
    "huggingface_config": {
      "split": "train",
      "column_mapping": {
        "question": "prompt",
        "answer": "label"
      }
    }
  }'
```

### Response

```json
{
  "id": "ds_hf123456789",
  "object": "dataset",
  "name": "gsm8k-train",
  "source_type": "hf",
  "source_uri": null,
  "huggingface_dataset": "gsm8k",
  "num_rows": null,
  "created_at": "2026-04-21T12:00:00Z"
}
```

## Validate Dataset Connection

This endpoint tests a Hugging Face source and returns preview metadata without creating a dataset row. Only `source_type: "hf"` is supported by the validator.

### Request

```bash
curl -X POST https://api.veri.studio/v1/datasets/connect/validate \
  -H "Authorization: Bearer vk_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "source_type": "hf",
    "huggingface_dataset": "openai/gsm8k",
    "huggingface_config": {
      "config_name": "main",
      "split": "train",
      "column_mapping": {"question": "prompt", "answer": "answer"}
    }
  }'
```

### Response

```json
{
  "valid": true,
  "num_rows": 1500,
  "columns": ["prompt", "expected_answer"],
  "error": null
}
```

## Dataset Format

Each uploaded JSONL row should be a JSON object with at least a `prompt` field:

```json
{"prompt": "What is 15 * 23?"}
{"prompt": "Solve for x: 2x + 5 = 17"}
{"prompt": "Explain the chain rule in calculus."}
```

The `prompt` value can be either a string or a list of chat messages. Additional fields stay available to reward functions and data resolvers.

<Note>
Connected datasets are resolved at job start, so validation is the best way to catch auth or schema issues early.
</Note>
