## Create Training Job

<ParamField body="base_model" type="string" required>
  HuggingFace model ID (e.g., `Qwen/Qwen3-4B`, `meta-llama/Llama-3.1-8B`).
</ParamField>

<ParamField body="dataset_id" type="string" required>
  ID of the uploaded dataset to train on.
</ParamField>

<ParamField body="reward_function_id" type="string" required>
  ID of the uploaded reward function.
</ParamField>

<ParamField body="output_name" type="string" required>
  Name for the output model checkpoint.
</ParamField>

<ParamField body="hyperparameters" type="object">
  Optional training hyperparameters.

  <Expandable title="hyperparameters">
    <ParamField body="learning_rate" type="float" default="1e-6">
      Learning rate for the optimizer.
    </ParamField>
    <ParamField body="num_epochs" type="integer" default="1">
      Number of training epochs.
    </ParamField>
    <ParamField body="max_steps" type="integer">
      Optional explicit training step limit.
    </ParamField>
    <ParamField body="rollouts_per_prompt" type="integer" default="8">
      Number of completions generated per prompt.
    </ParamField>
    <ParamField body="kl_coef" type="float" default="0.001">
      KL penalty coefficient.
    </ParamField>
    <ParamField body="max_prompt_length" type="integer" default="1024">
      Prompt token cap.
    </ParamField>
    <ParamField body="max_response_length" type="integer" default="2048">
      Completion token cap.
    </ParamField>
    <ParamField body="global_batch_size" type="integer" default="64">
      Global batch size.
    </ParamField>
    <ParamField body="seed" type="integer" default="42">
      Random seed.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="gpu" type="object">
  Optional requested GPU configuration.
</ParamField>

<ParamField body="checkpoint_destination" type="object">
  Optional final checkpoint target. Defaults to `{ "type": "veri" }`.
</ParamField>

### Request

```bash
curl -X POST http://localhost:8000/v1/training_jobs \
  -H "Authorization: Bearer $VERI_INTERNAL_API_KEY" \
  -H "X-Veri-User-Id: $VERI_USER_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "base_model": "Qwen/Qwen3-4B",
    "dataset_id": "ds_abc123",
    "reward_function_id": "rf_def456",
    "output_name": "qwen3-4b-math",
    "hyperparameters": {
      "learning_rate": 1e-6,
      "num_epochs": 1,
      "rollouts_per_prompt": 8,
      "max_response_length": 2048,
      "global_batch_size": 64
    },
    "gpu": {
      "gpu_type": "A100-80GB",
      "gpu_count": 1
    },
    "checkpoint_destination": {
      "type": "veri"
    }
  }'
```

## List Training Jobs

<ParamField query="limit" type="integer" default="20">
  Maximum number of jobs to return.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of jobs to skip.
</ParamField>

<ParamField query="status" type="string">
  Filter by job status: `queued`, `provisioning`, `running`, `completed`, `failed`, `cancelled`.
</ParamField>

### Request

```bash
curl "http://localhost:8000/v1/training_jobs?status=running&limit=10" \
  -H "Authorization: Bearer $VERI_INTERNAL_API_KEY" \
  -H "X-Veri-User-Id: $VERI_USER_ID"
```

### Response

```json
{
  "items": [
    {
      "id": "job_xyz789",
      "status": "running",
      "base_model": "Qwen/Qwen3-4B",
      "output_name": "qwen3-4b-math",
      "dashboard_url": "http://localhost:3001/d/veri-training?var-job_id=job_xyz789",
      "created_at": "2026-04-14T12:10:00Z",
      "started_at": "2026-04-14T12:11:00Z"
    }
  ],
  "total": 1,
  "limit": 10,
  "offset": 0
}
```

### Job Statuses

| Status | Description |
|--------|-------------|
| `queued` | Job row created and waiting to be submitted |
| `provisioning` | Backend accepted the job and compute is starting |
| `running` | Training is in progress |
| `completed` | Training finished successfully; checkpoint is available |
| `failed` | Training failed; check logs for details |
| `cancelled` | Job was cancelled by the user |

## Cancel Training Job

### Request

```bash
curl -X DELETE http://localhost:8000/v1/training_jobs/job_xyz789 \
  -H "Authorization: Bearer $VERI_INTERNAL_API_KEY" \
  -H "X-Veri-User-Id: $VERI_USER_ID"
```

### Response

```json
{
  "id": "job_xyz789",
  "status": "cancelled"
}
```

<Note>
Only non-terminal jobs can be cancelled. If the job is already `completed`, `failed`, or `cancelled`, the API returns `400`.
</Note>

## Stream Job Logs

Returns job logs as Server-Sent Events (SSE).

<Note>
Real-time log streaming from training is not yet implemented. Currently this endpoint returns a link to the Modal dashboard where logs can be viewed. Full SSE streaming of training progress (step, loss, reward) is planned.
</Note>

### Request

```bash
curl -N https://api.veri.studio/v1/training_jobs/job_xyz789/logs \
  -H "Authorization: Bearer vk_your_api_key"
```

### Response (SSE stream)

```
data: Logs available at: https://modal.com/calls/fc-01KPQ...
```

## Get Model Checkpoint

Returns a download URL for the trained model checkpoint. Only available for jobs with `completed` status.

### Request

```bash
curl http://localhost:8000/v1/training_jobs/job_xyz789/model \
  -H "Authorization: Bearer $VERI_INTERNAL_API_KEY" \
  -H "X-Veri-User-Id: $VERI_USER_ID"
```

### Response

```json
{
  "download_url": "https://storage.veri.studio/checkpoints/job_xyz789/model.tar.gz"
}
```

<Note>
The same URL is also surfaced as `download_url` on completed job responses when checkpoint resolution succeeds.
</Note>
