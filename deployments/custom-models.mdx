A custom model is a set of weights you own, stored on Veri and deployable from a single named entry. Every custom model lands in the same place (Veri-managed storage) no matter how it got there, so deploying one is always the same one step regardless of where the weights came from.

Your custom models live in the **model library** in the dashboard. From there you deploy, download, or copy a model's ID for the API.

## Where custom models come from

| Source | What it is | How it gets there |
| --- | --- | --- |
| Training job | A checkpoint from a completed Veri training job | Already on Veri storage; add it to your library with one click |
| Hugging Face import | Any model on the Hugging Face Hub, including private and gated repos | Veri pulls the weights server-side into your library |
| Direct upload | Weights from your machine | Upload through the dashboard |

Whatever the source, the result is one library entry whose weights live on Veri. When you deploy it, there is no second download from an outside host at serve time.

## From a training job

When a training job completes, its checkpoint is already on Veri-managed storage. Add it to your library to give it a name and make it deployable like any other model.

You can also keep working with the raw checkpoint directly:

```bash
veri jobs download <job-id> --output-dir ./checkpoint
```

See [Fine-tuning](/fine-tuning) for the full training flow.

## Import from Hugging Face

Importing copies the weights from the Hugging Face Hub into your Veri library. The download runs on Veri, not on your machine, so a large model never round-trips through your laptop.

```bash
veri models import Qwen/Qwen2.5-0.5B-Instruct --name qwen-base
```

The import runs as a background job. Track it with:

```bash
veri models list
```

The model is `importing` while the weights copy, then `ready` once it can be deployed.

### Private and gated models

For a private or gated repo, pass a Hugging Face access token. Veri uses the token only to authenticate the one-time download and does not store it.

```bash
veri models import meta-llama/Llama-3.1-8B-Instruct \
  --name llama-3-8b \
  --hf-token "$HF_TOKEN"
```

<Note>
  Use a [fine-grained read token](https://huggingface.co/docs/hub/security-tokens) scoped to the repos you import. Gated models (for example, Llama) require that you have already accepted the model's license on Hugging Face with the same account the token belongs to.
</Note>

## Upload from your machine

Upload weights directly through the dashboard. The model must use the standard Hugging Face layout: `config.json`, the weight files (`.safetensors`), and the tokenizer files.

<Note>
  Direct upload through the dashboard is sized for small models and LoRA adapters. For anything larger, import from Hugging Face instead. The server-side pull is faster and does not depend on your connection. CLI upload is on the roadmap.
</Note>

## Model states

| State | Meaning |
| --- | --- |
| `importing` | Weights are copying into your library. Not yet deployable. |
| `ready` | Weights are stored and the model can be deployed. |
| `failed` | The import or upload did not finish. The error explains why (for example, a gated repo without a valid token). |

## Size limits

A custom model must fit on a single GPU node, the same ceiling as a deployment. Pick a [GPU large enough to hold the weights](/deployments/hosting) when you deploy.

## Deploy a custom model

Once a model is `ready`, deploy it the same way you deploy any model: pass its ID. Custom models always serve from Veri storage, so cold starts skip the external download step.

```python
dep = client.deployments.create(
    model="<your-model-id>",
    name="my-deploy",
    gpu={"gpu_type": "A100-80GB", "gpu_count": 1},
)
```

See [Deployments](/deployments) for the serving lifecycle and the OpenAI-compatible chat surface.

## Getting started without a model

If your library is empty, you have not trained or imported anything yet. Two ways to start:

- Browse the model library in the dashboard and import a base model to deploy right away.
- [Fine-tune your own model](/fine-tuning), then add the resulting checkpoint to your library.
