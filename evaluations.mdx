## What evaluations will do

Evaluations will let you score a deployed model against a held-out dataset and read both an aggregate score and per-sample results, so you can see exactly which inputs a model gets wrong.

## Planned scorers

- `exact_match`: case-insensitive comparison against a dataset reference field.
- `contains`: substring or regex match against a reference field.
- `llm_judge`, custom `python` scorers, and curated public benchmarks (MMLU, GPQA, HumanEval, and similar) are planned for later stages.

The SDK, CLI, and API surface for evaluations will be documented here once the feature ships.
