This walks through serving a model end-to-end: create a deployment, wait for it to be ready, send chat requests, then stop it so you don't keep paying.

We use **Qwen2.5-0.5B-Instruct** so it boots quickly. Swap in any HF model name once you've seen it work. You may also pass a training job ID instead to serve your own trained checkpoint.

<Warning>
  This quickstart spends real GPU credit. Deployments charge by the hour while they're running, and there's no idle auto-stop. Check your balance with `veri billing balance` before you start, and remember to stop the deployment at the end.
</Warning>

<Steps>
  <Step title="Prereq: log in">
    If you haven't already, finish the [Quickstart](/quickstart) so `veri login` is done.

    ```bash
    veri whoami
    ```
  </Step>

  <Step title="Create a deployment">
    <Tabs>
      <Tab title="CLI">
        ```bash
        veri deployments create \
          --from-hf Qwen/Qwen2.5-0.5B-Instruct \
          --name my-first-deploy \
          --gpu-type A100-80GB \
          --gpu-count 1
        ```

        Copy the printed deployment ID and export it for the rest of this page:

        ```bash
        export DEP_ID=dep_abc        # paste the id you got back
        ```

        To serve your own trained model from a completed training job, swap `--from-hf` for `--model <job-id>`.
      </Tab>
      <Tab title="Python SDK">
        ```python
        from veri_sdk import Client
        client = Client()

        dep = client.deployments.create(
            model="Qwen/Qwen2.5-0.5B-Instruct",
            source="huggingface",
            name="my-first-deploy",
            gpu={"gpu_type": "A100-80GB", "gpu_count": 1},
        )

        print(f"Deployment {dep.id} — status: {dep.status}")
        ```

        To serve your own trained model, pass `model=job.id, source="training_job"` instead.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Wait for serving">
    A 0.5B model typically reaches `serving` in 1 to 3 minutes. Larger models take longer to pull weights.

    <Tabs>
      <Tab title="CLI">
        ```bash
        until veri deployments get $DEP_ID --format json | jq -r .status | grep -q serving; do
          sleep 10
        done

        veri deployments get $DEP_ID --format json | jq -r .endpoint_url
        ```
      </Tab>
      <Tab title="Python SDK">
        ```python
        dep.wait()
        print(f"Endpoint: {dep.endpoint_url}")
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Chat with it">
    <Tabs>
      <Tab title="CLI">
        ```bash
        veri deployments chat $DEP_ID --message "What is 12 + 7?"
        ```
      </Tab>
      <Tab title="Python SDK">
        ```python
        response = dep.chat([
            {"role": "user", "content": "What is 12 + 7?"},
        ])

        print(response["choices"][0]["message"]["content"])
        ```

        The response payload is OpenAI-compatible (same shape as `chat.completions.create`).
      </Tab>
    </Tabs>
  </Step>

  <Step title="Call it as an OpenAI endpoint">
    Each deployment exposes an OpenAI-compatible `/chat/completions` URL, so any tool that speaks OpenAI (LangChain, LlamaIndex, Vercel AI SDK, your existing code) works without changes.

    <Tabs>
      <Tab title="curl">
        ```bash
        ENDPOINT=$(veri deployments get $DEP_ID --format json | jq -r .endpoint_url)

        curl -s "$ENDPOINT/chat/completions" \
          -H "Authorization: Bearer $VERI_API_KEY" \
          -H "Content-Type: application/json" \
          -d '{
            "model": "my-first-deploy",
            "messages": [{"role": "user", "content": "What is 12 + 7?"}]
          }' | jq .choices[0].message.content
        ```
      </Tab>
      <Tab title="OpenAI Python SDK">
        ```python
        from openai import OpenAI

        oai = OpenAI(
            base_url=dep.endpoint_url,
            api_key="vk_your_key_here",
        )

        response = oai.chat.completions.create(
            model=dep.name,
            messages=[{"role": "user", "content": "What is 12 + 7?"}],
        )

        print(response.choices[0].message.content)
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Stop the deployment">
    <Warning>
      Don't skip this. Idle deployments accrue cost.
    </Warning>

    <Tabs>
      <Tab title="CLI">
        ```bash
        veri deployments stop $DEP_ID

        veri deployments get $DEP_ID --format json \
          | jq '{total_requests}'
        ```
      </Tab>
      <Tab title="Python SDK">
        ```python
        dep.stop()

        final = client.deployments.get(dep.id)
        print(f"Requests served: {final.total_requests}")
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>
