This walks through serving a model end-to-end: create a deployment, wait for it to be ready, send chat requests via both the Veri SDK and the OpenAI SDK, then stop it so you don't keep paying.

We use **Qwen2.5-0.5B-Instruct** so it boots quickly. Swap in any HF model name once you've seen it work — or pass a training job ID instead to serve your own fine-tuned checkpoint.

<Warning>
  This quickstart spends real GPU credit. Deployments charge by the hour while they're running, and there's no idle auto-stop. Check your balance with `veri billing balance` before you start, and make sure you call `dep.stop()` at the end.
</Warning>

<Steps>
  <Step title="Prereq: log in">
    If you haven't already, finish the [Quickstart](/quickstart) — you need an installed SDK and `veri login` done.

    ```python
    from veri_sdk import Client
    client = Client()
    ```
  </Step>

  <Step title="Create a deployment">
    ```python
    dep = client.deployments.create(
        model="Qwen/Qwen2.5-0.5B-Instruct",
        source="huggingface",
        name="my-first-deploy",
        gpu={"gpu_type": "A100-80GB", "gpu_count": 1},
    )

    print(f"Deployment {dep.id} — status: {dep.status}")
    ```

    To serve your own fine-tuned model from a completed training job, swap to:

    ```python
    dep = client.deployments.create(
        model=job.id,
        source="training_job",
        name="my-fine-tuned-model",
        gpu={"gpu_type": "A100-80GB", "gpu_count": 1},
    )
    ```
  </Step>

  <Step title="Wait for serving">
    ```python
    dep.wait()
    print(f"Endpoint: {dep.endpoint_url}")
    ```

    A 0.5B model typically reaches `serving` in 1–3 minutes. Larger models take longer to pull weights.
  </Step>

  <Step title="Chat with it (Veri SDK)">
    ```python
    response = dep.chat([
        {"role": "user", "content": "What is 12 + 7?"},
    ])

    print(response["choices"][0]["message"]["content"])
    ```

    The response payload is OpenAI-compatible — same shape as `chat.completions.create`.
  </Step>

  <Step title="Chat with it (OpenAI SDK)">
    Because the endpoint is OpenAI-compatible, you can point the official OpenAI SDK at it:

    ```python
    from openai import OpenAI

    oai = OpenAI(
        base_url=dep.endpoint_url,
        api_key="vk_your_key_here",
    )

    response = oai.chat.completions.create(
        model=dep.name,
        messages=[{"role": "user", "content": "What is 12 + 7?"}],
    )

    print(response.choices[0].message.content)
    ```

    This means any tool that talks OpenAI — LangChain, LlamaIndex, Vercel AI SDK, your own existing code — works against a Veri deployment without changes.
  </Step>

  <Step title="Stop the deployment">
    <Warning>
      Don't skip this. Idle deployments accrue cost.
    </Warning>

    ```python
    dep.stop()

    # Refresh and read final cost
    final = client.deployments.get(dep.id)
    print(f"Total cost: ${final.total_cost_usd:.4f}")
    print(f"Requests served: {final.total_requests}")
    ```
  </Step>
</Steps>
