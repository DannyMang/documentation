Training is **one decorator**. Define a reward function, decorate it with your training config, and run with `veri train`.

We use **Qwen2.5-0.5B-Instruct** for a quick GRPO run on [GSM8K](https://huggingface.co/datasets/openai/gsm8k). The 0.5B / 50-step path fits on a single **L4** (24 GB) for a fast smoke test.

New to GRPO (Group Relative Policy Optimization)? [Here's a quick read](https://unsloth.ai/blog/r1-reasoning).

<Warning>
  This quickstart spends real GPU credit. Check your balance with `veri billing balance` before you start.
</Warning>

## Write your script

Save as `train.py`:

```python
from veri_sdk import training_job


@training_job(
    base_model="Qwen/Qwen2.5-0.5B-Instruct",
    dataset="hf:openai/gsm8k",
    dataset_config="main",
    column_mapping={"question": "prompt", "answer": "label"},
    gpu_type="L4-24GB",
    gpu_count=1,
    hyperparameters={
        "learning_rate": 1e-6,
        "rollouts_per_prompt": 4,
        "max_response_length": 512,
        "kl_coef": 0.001,
        "max_steps": 50,
    },
)
def reward(completions, answer, **kwargs):
    import re

    scores = []
    for c, expected in zip(completions, answer):
        text = c[-1]["content"] if isinstance(c, list) else str(c)
        format_ok = bool(re.search(r"<answer>.*?</answer>", text, re.DOTALL))
        m = re.search(r"<answer>(.*?)</answer>", text, re.DOTALL)
        correct = bool(m and expected.strip() in m.group(1).strip())
        scores.append(0.5 * float(format_ok) + 0.5 * float(correct))
    return scores
```

The `@training_job` decorator is **metadata only**. Importing this file does nothing at runtime. `veri train` is what actually uploads, submits, and waits.

## Run it

```bash
veri train train.py
```

You'll see:


## Download the checkpoint

The script doesn't auto-download. Once it prints `status: completed`, pull the checkpoint:

<CodeGroup>
```bash CLI
veri jobs download tj_def789 --output-dir ./checkpoints
```

```python SDK
from veri_sdk import Client

client = Client()
job = client.training_jobs.get("tj_def789")
job.download("./checkpoints")
```
</CodeGroup>

## Changeable parameters

Every kwarg on `@training_job` is documented in the [API reference](/api-reference/training-jobs). Here are some you might want to change:

| Kwarg | What it controls |
|---|---|
| `base_model` | Any Hugging Face causal LM. Qwen2.5-7B is a good production default once your loop works. |
| `dataset` | `"hf:<org>/<name>"` to auto-connect a Hugging Face dataset, or `"ds_<id>"` to reference one you already uploaded. |
| `hyperparameters` | `learning_rate`, `rollouts_per_prompt`, `max_steps`, `kl_coef`, `max_response_length`, plus method-specific keys. See [GRPO](/training/grpo). |
| `gpu_type` / `gpu_count` | `L4-24GB` (1 GPU) fits ~0.5B-3B. Bigger models need `A100-80GB` or `H100-80GB`, which AWS ships as 8-GPU nodes (`gpu_count=8`). Run `veri gpu list` to see options. |
