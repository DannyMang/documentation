A scorer turns a model output into a number. An eval can have any number of scorers — each one runs against every dataset row and contributes its own column to the aggregate.

## Available now (Phase A)

### `exact_match`

```json
{
  "type": "exact_match",
  "name": "exact_answer",
  "reference_field": "answer",
  "case_sensitive": false
}
```

| Field | Default | Notes |
| --- | --- | --- |
| `name` | required | Appears as a key in `run.results` |
| `reference_field` | `"label"` | Which dataset column holds the ground truth |
| `case_sensitive` | `false` | When `false`, lowercase both before comparing |

**Semantics:** `output.strip() == item[reference_field].strip()` (with case conversion if needed). Returns `1.0` for a match, `0.0` otherwise.

**Edge cases:**
- Missing reference field → `0.0` (not an error)
- Empty output → `0.0`

### `contains`

```json
{
  "type": "contains",
  "name": "has_answer",
  "reference_field": "answer",
  "regex": false
}
```

| Field | Default | Notes |
| --- | --- | --- |
| `name` | required | Result key |
| `reference_field` | `"label"` | Source column for the reference value |
| `regex` | `false` | Treat the reference as a regex pattern |

**Semantics:**
- `regex=false` (default): substring check. `1.0` if `reference in output`.
- `regex=true`: `pattern.search(output)`. Compiled patterns are cached.

**Edge cases:**
- Invalid regex pattern → logged once, treated as `0.0`
- Missing reference field → `0.0`

## Coming soon (Phase B / C)

These types are accepted by the schema today but rejected at run-time. They ship in the order below.

### `reward_function`

```json
{
  "type": "reward_function",
  "name": "math_reward",
  "reward_function_id": "rf_abc123"
}
```

Loads a previously-uploaded TRL or Miles reward function and scores each row with it. Runs in a subprocess sandbox with a 10-second timeout per item.

### `llm_judge`

```json
{
  "type": "llm_judge",
  "name": "helpfulness",
  "judge_prompt": "Rate this answer 1-5 for helpfulness: {output}",
  "judge_model": "deployment:dep_judge_model",
  "judge_temperature": 0.0
}
```

Calls a configurable judge model with a templated prompt. Default judge will be a shared Veri-hosted open model; BYOK frontier comparators (`openai:gpt-4o`, `anthropic:claude-...`) come with the BYOK integration.

### `python`

```json
{
  "type": "python",
  "name": "custom_score",
  "source_code": "def score(input, output, item) -> float: ..."
}
```

Arbitrary scoring function in a subprocess sandbox. Lower friction than `reward_function` for one-off evals.

### `benchmark`

```json
{
  "type": "benchmark",
  "name": "mmlu_baseline",
  "task": "mmlu_0_shot",
  "sample_limit": 500
}
```

Inspect AI-backed standard benchmarks. Whitelisted tasks: `mmlu_0_shot`, `mmlu_pro`, `gpqa_diamond`, `humaneval`, `mbpp`, `ifeval`, `math_500`, `gsm8k`, `aime2024`, `aime2025`, `hellaswag`, `arc_challenge`, `truthfulqa`.

## Using multiple scorers

Define them in a list — each one's score appears as a separate key in results.

```python
client.evals.create(
    name="gsm8k-multi-signal",
    dataset_id=dataset.id,
    scorers=[
        {"type": "exact_match", "name": "exact", "reference_field": "answer"},
        {"type": "contains", "name": "mentions", "reference_field": "answer"},
    ],
)
```

After a run completes, `run.results` looks like:

```json
{
  "total_items": 100,
  "scores": {
    "exact": {"mean": 0.42, "median": 0.0, "std": 0.49, "min": 0.0, "max": 1.0, "count": 100, "error_count": 0},
    "mentions": {"mean": 0.71, "median": 1.0, "std": 0.45, "min": 0.0, "max": 1.0, "count": 100, "error_count": 0}
  }
}
```

## Scorer error handling

If a scorer raises an exception on a particular item, the score is recorded as `null` for that item and the worker continues. The aggregate counts these in `error_count` per scorer; the rest of the items still produce a valid `mean`.

For Phase A scorers, errors are extremely rare (only invalid regex on `contains` triggers them).
