## What scorers will do

A scorer turns a model's output for a single dataset item into a numeric score. An evaluation run applies one or more scorers across every item, then aggregates the per-item scores.

## Planned scorer types

- `exact_match`: case-insensitive comparison against a dataset reference field.
- `contains`: substring or regex match against a reference field.
- `llm_judge`: grade outputs with a judge model.
- `python`: bring your own scoring function.
- Curated public benchmarks (MMLU, GPQA, HumanEval, and similar) in a later stage.

Exact configuration and the API will be documented here once evaluations ship.
